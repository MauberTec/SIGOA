Módulo de Anomalias

 

O módulo de Anomalias pode ser acessado no menu lateral

Legendas

 

Primeiro item do módulo de Anomalias

Os tipos de Anomalias são agrupados em Legendas. Para o sistema a legenda é a sigla que antecede o número indicativo do tipo da anomalia (por exemplo F para fissuras).

Para cada Legenda estão definidos símbolos e hachuras para identificação destas anomalias nos croquis que são apresentados nos Relatórios de Inspeção como um dos produtos da inspeção.

 

Observação: Todas as Legendas previstas já estarão carregadas ao ser implantado o sistema.

 

 

Tipos de Anomalia

 

Segundo item do módulo de Anomalias

Para cada Legenda de Anomalia, são definidos os Tipos das Anomalias.

 

Observação: Assim como as Legendas, todos os tipos de Anomalias estarão todos cadastrados quando o sistema for implantado.

 

Causas

 

O Terceiro item do módulo de Anomalias

Para cada Legenda de Anomalia, são definidas as Causas prováveis para a anomalia detectada.

 

Observação: As Causas de Anomalias também estarão cadastradas quando o sistema for implantado.

 

 

Níveis de Alerta

 

Quarto item do módulo de Anomalias

Todas as Anomalias detectadas em uma inspeção, são classificadas quanto ao nível de criticidade aqui definido como Nível de Alerta.

 

 

Observação: Os Níveis de Alerta estabelecidos pela Engenharia estarão cadastradas quando o sistema for implantado.

 

 

Status

 

Quinto item do módulo de Anomalias

Os Status previstos para as Anomalias são: Aguardando Reparo, Reparada e Superada.

 

 

Observação: Quando uma OS de Inspeção Especial é encerrada, todas as Anomalias detectadas com indicação de Reparos, terão seus status preenchidos como "Aguardando Reparo" para que seja feito o acompanhamento do seu estado até que ela seja reparada.

 

Fluxo de Status

 

Sexto item do módulo de Anomalias

Uma Anomalia Aguardando Reparo poderá ter o seu status atualizado para Reparada ou Superada.

As Anomalias detectadas em uma Inspeção Especial, são associadas à OS de Inspeção Especial. Estes registros são transferidos para as OS's de Orçamento e na sequencia para as OS's de Reparo.

Os status das Anomalias vão sendo alterados, a medida que uma OS de Orçamento é aprovada ou uma OS de Reparo é aberta ou executada.

Nestas ocasiões, o status na OS anterior (OS de Inspeção Especial no caso de uma OS de Orçamento aprovada, ou OS de Orçamento no caso de uma OS de Reparo Planejada) é atualizado para SUPERADO, e o dados são copiados para a próxima OS (OS de Orçamento ou OS de Reparo) com o status da anomalia como AGUARDANDO REPARO, até que a OS de Reparo seja executada e a anomalia seja reparada.
